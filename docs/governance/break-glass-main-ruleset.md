# Break-Glass Main Ruleset

Normal direct pushes, force pushes, deletion and bypass are prohibited on
`main`. Changes require a PR, one independent approval, code-owner review,
last-push approval, resolved threads and all required checks.

An emergency must be documented in an incident record before a repository
administrator temporarily changes the ruleset. Record actor, reason, exact SHA,
time window and rollback. Restore the ruleset immediately and obtain a
retrospective independent review. There is no standing bypass actor.
